思考题回答

根据提供的讲义内容，以下是针对第6章思考题的回答：

1. 系统的第一个线程是什么？初始化它的时候为何不调用Setup_Kernel_Thread？

回答：

  • 系统的第一个线程是 Main 函数。 这个函数是内核的入口点，负责初始化各种子系统，并启动其他内核线程。
  • 初始化 Main 线程时不调用 Setup_Kernel_Thread 的原因：
    • Main 函数是内核的第一个线程，它的上下文和堆栈已经在引导加载程序中设置好了。因此，不需要通过 Setup_Kernel_Thread 来设置其上下文。
    • 在 Main 函数执行时，许多内核子系统（如内存管理、调度器等）还没有完全初始化。如果在这个阶段调用 Setup_Kernel_Thread，可能会导致依赖问题。
    • Main 函数直接控制了内核的启动流程，包括初始化各个子系统和创建其他内核线程。这种控制流确保了所有初始化步骤按顺序执行，避免了并发带来的复杂性。

2. 内核线程结构体中定义了三组指针，它们的作用是什么？

回答：

  • 内核线程结构体 Kernel_Thread 中定义的三组指针及其作用：
    • userContext 指针：指向用户态进程上下文。对于内核线程，这个指针为空；对于用户态进程，这个指针指向 User_Context 结构体，用于存储用户态进程的相关信息。
    • stackPage 指针：指向该线程的堆栈页。每个线程都有自己的堆栈空间，用于保存局部变量、函数参数和返回地址等。
    • next 和 prev 指针：用于将线程链接到链表中。这些指针使得线程可以被组织成链表结构，便于管理和调度。例如，s_runQueue 是一个运行队列，s_allThreadList 是所有线程的链表。

3. 区别 Setup_Kernel_Thread 中的 Push 函数与汇编指令 push

  • Push 函数：

    • Push 函数是 C 语言中的一个函数，位于 kthread.c 文件中。它通过操作线程结构体内部的 esp 指针来模拟对线程堆栈的操作。
    • 具体实现如下：
    • Push 函数主要用于在设置新线程的上下文时，将一些初始值压入线程的堆栈中。

  • **汇编指令 push**：

    • push 是 x86 汇编语言中的一个指令，用于将数据压入当前执行线程的堆栈中。
    • 例如，在 Switch_To_Thread 函数中，push 指令用于保存当前线程的上下文信息，以便稍后恢复。

4. Switch_To_Thread 中调整堆栈的目的是什么？

回答：

  • 调整堆栈的目的：
    • 保存当前线程的上下文：在切换到新的线程之前，需要保存当前线程的上下文信息，包括寄存器状态、堆栈指针等。这通常是通过将这些信息压入堆栈来完成的。
    • 为新线程准备上下文：调整堆栈以使新线程能够正确地开始执行。这包括将新线程的堆栈指针设置为正确的值，以及将新线程的上下文信息从堆栈中恢复。
    • 模拟中断返回：Switch_To_Thread 通过调整堆栈，使其看起来像是从一个中断返回。这样，当新线程开始执行时，它会从正确的地址继续执行，而不会丢失任何上下文信息。

具体来说，Switch_To_Thread 中的堆栈调整步骤包括：

  • 保存当前线程的 eax 寄存器。
  • 获取并调整返回地址。
  • 将 eflags 和 cs 寄存器的值压入堆栈。
  • 保存通用寄存器的状态。
  • 设置新线程的堆栈指针。
  • 恢复新线程的上下文信息。

通过这些步骤，Switch_To_Thread 确保了线程切换过程中的上下文完整性，使得新线程能够顺利地从上次暂停的地方继续执行。
