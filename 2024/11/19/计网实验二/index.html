Scapy是一个 Python程序，它允许用户发送、嗅探、分析和伪造网络包。这种能力允许构建能够探测、扫描或攻击网络的工具。scapy还可以很好地执行许多其他工具无法处理的其他特定任务，例如发送无效帧、注入自己的802.11帧、结合技术（VLAN跳跃+ARP缓存中毒、WEP加密通道上的VoIP解码等）。

原理

回忆一下以太网数据帧的组成

目的MAC地址 源MAC地址 类型 数据 FCS
6字节 6字节 2字节 [46,1500] 4字节

现在解释一下实验指导书提到的以太网帧的填充字段。以太网帧中的PAD（填充）字段。以太网帧的结构主要包括以下几个部分：

  • 前导码（Preamble）：56位（7字节），用于接收同步阶段。
  • 目的地址（Destination Address）：48位（6字节）。
  • 源地址（Source Address）：48位（6字节）。
  • 类型/长度（Type/Length）：16位（2字节），当值小于或等于1500时，表示长度字段；大于1536时，表示类型字段。
  • 数据（Data）：46到1500字节。这是PDU（协议数据单元）所在的位置。
  • 填充（Padding）：0到46字节，确保整个帧的最小长度为64字节。如果数据部分小于46字节，则使用PAD来填充。
  • 帧校验序列（Frame Check Sequence, FCS）：32位（4字节），用于错误检测。
  1. 根据描述，以太网帧的最小长度是64字节，这包括了18字节的固定开销（目的地址6字节 + 源地址6字节 + 类型/长度2字节 + FCS 4字节）加上至少46字节的数据或数据加填充。
  2. 如果实际的数据部分（PDU）不足46字节，那么就需要通过PAD字段来补充，以保证整个帧的长度不低于64字节。这是因为以太网物理层设计中存在一个最小帧时间的要求，低于这个时间的帧可能会被视为无效或者噪声引起的错误帧，即所谓的“帧碎片”，通常会被网络设备直接丢弃而不进行进一步处理。因此，正确的填充是保证以太网通信可靠性的必要措施之一。

FCS（Frame Check Sequence，帧校验序列）是一种用于检测数据传输过程中错误的技术。在以太网和其他网络协议中，FCS 通常使用 32 位的 CRC（Cyclic Redundancy Check，循环冗余校验）来实现。CRC 是一种非常有效的错误检测机制，能够检测出大多数常见的传输错误。

生成多项式 G(X) 和 CRC-32

1. 生成多项式 G(X)

生成多项式（Generator Polynomial）是一个特定的多项式，用于生成 CRC 校验码。对于 32 位的 CRC 校验码，最常用的生成多项式是 G(X) = X^32 + X^26 + X^23 + X^22 + X^16 + X^12 + X^11 + X^10 + X^8 + X^7 + X^5 + X^4 + X^2 + X + 1。

这个多项式可以表示为二进制数 100000100110000010001110110110111，或者十六进制数 0x04C11DB7。

2. CRC-32

CRC-32 是一种使用 32 位生成多项式的 CRC 校验算法。它是最常用的一种 CRC 校验算法，广泛应用于各种通信协议和文件格式中。CRC-32 的生成多项式就是上面提到的 0x04C11DB7。

CRC-32 的计算过程

  1. 初始值：通常，CRC-32 的初始值设为全 1（即 0xFFFFFFFF），但也可以根据具体应用设置为其他值。
  2. 预处理：将数据帧的每个字节依次处理。
  3. 多项式除法：使用生成多项式 0x04C11DB7 对数据进行模 2 除法运算。
  4. 结果：最终得到的 32 位结果就是 CRC-32 校验码。
